Android屏幕适配

1、pt,px,dp

1. pt(point):磅/点,固定长度单位,1pt = 1/72inch = 2.54cm
2. px(pixel):像素,相对大小单位

2、换算

->1pt = (ppi / 72)px

3、屏幕尺寸(单位:英寸)

1. 屏幕对象线的长度
2. Android把所有的屏幕大小分为四种尺寸:小,普通,大,超大
    - 小:一般指屏幕尺寸小于3英寸左右
    - 中:一般指屏幕尺寸小于4.5英寸左右
    - 大:一般指屏幕尺寸在4.5英寸~7英寸左右
    - 超大:一般指屏幕尺寸7英寸~10英寸左右

4、分辨率

1. 判断屏幕清晰度的指标:ppi,每英寸所拥有的像素点/每英寸像素
2. 计算公式:√(2X+2Y) / Z,其中X:垂直像素数;Y:水平像素数,Z:屏幕尺寸
    - 可在百度等搜索引擎中搜索ppi计算,如:http://www.lukiya.com/App/PPICalculator
3. PS:当你拿的手机距离你约25~30厘米,如果手机ppi达到300以上,你的眼睛将无法分辨出像素点.这也就是苹果提出的视网膜屏幕

5、dpi

- 印刷单位,指每英寸所能印刷出来的网点数

6、dp,px转换

1. dp(Density-independent Pixels)
    - 与屏幕无关的像素单位.Android开发的虚拟单位.在每英寸160点的显示器上,1dp = 1px
2.dip(Device Independent Pixels)
    - 设备独立像素单位,Android开发虚拟单位,多用于Android示例计数单位
3. 在Android开发中,dp=dip
4. sp(Scale-independent Pixels)
    - Android字体大小虚拟单位.在每英寸160点的显示器上,当文字大小为100%时,1sp = 1px
5. 单位换算:
    
    - dp = (160 / ppi) * px
    - px = dp * ppi / 160

7. UI屏幕适配方案

1. 适配原则
    - 尽可能适配大部分用户 
    - 能实现承上启下
    - 符合设备发展趋势
    - 经济适用
2. 查看当前手机分辨率占有率状况
    - 进入http://www.umindex.com/devices/android_resolutions 查看当前手机分辨率市场占用率情况
3. 让不同的内容自动适应不同的屏幕比例
    - 在layout文件夹分别对layout-w360dp和layout-w320dp调整布局
    - 多套分辨率的切图,如nodpi,xhdpi,xxhdpi等
        - nodpi是指在任何分辨率的屏幕上显示都是一样的大小
    - 建议提供一套XXHdpi的图片.对于图标等要求净系的图片.可以再制作一套XHdpi补充.
    - 如果考虑横竖屏的布局效果.Android有layout-land和layout-port两个目录一.分别放置横竖屏两种布局文件.在手机屏幕方向变化时会自动调用相应的布局文件
4. 预测未来几年的主流分辨率为1080P
